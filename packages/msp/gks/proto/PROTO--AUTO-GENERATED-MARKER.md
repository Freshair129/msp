# PROTO — AUTO-GENERATED-MARKER

## Rule

Every file produced by Phase 3.5 micro-task codegen MUST begin with the canonical AUTO-GENERATED marker block before any other content (`FRAMEWORK_MASTER_SPEC.md` §9.6):

```
// AUTO-GENERATED — DO NOT EDIT
// Source: .brain/<ns>/tasks/<feature>/manifest.yaml
// Regenerate: npm run msp:run-task <feature>
```

The comment prefix (`//`, `#`, `--`, etc.) is the only language-specific knob; the three-line content is fixed.

## Schema

| Line | Required content |
|---|---|
| 1 | `<prefix>AUTO-GENERATED — DO NOT EDIT` |
| 2 | `<prefix>Source: <relative-path-to-manifest>` |
| 3 | `<prefix>Regenerate: npm run msp:run-task <feature-id>` |

## Why

Without the marker, hand edits to generated files survive a regenerate-and-overwrite cycle silently. §9.6 elevates the marker to a machine-enforced invariant so editors and CI both have a deterministic signal.

## Validator

Severity: `warning` while Phase 0 of the cognitive-layer rollout lands; flip to `error` once the deterministic composer is in production.

Implemented by `buildAutoGeneratedMarker()` and `composeWithMarker()` in `packages/msp/src/cognitive/compose.ts`; the marker tag itself is exported from `marker-constants.ts` so the validator can reference it.

## Exceptions

Files inside `gks/` (atoms) are author-edited and never carry the marker.
Files under `src/` that pre-date Phase 0 codegen are grandfathered; the marker is only required for files emitted by `msp:run-task` after this PROTO promotes to `stable`.
